# Progress Log - 2026-04-16

## Summary

Today we turned the initial jewellery storefront scaffold into a cleaner product-first experience for Rouhi Gems and started wiring live gold pricing support.

## What Changed

### Frontend direction

- Reworked the homepage, catalogue, and about-page copy so it reads like a live customer-facing brand instead of an internal build note.
- Removed the generic custom-order-first approach from the user experience.
- Kept `/custom-order` only as a redirect to `/catalogue` so the old flow is not exposed.

### Product-first customisation flow

- Moved customisation onto the product detail page.
- Each piece now has fixed:
  - gold weight
  - diamond weight
  - diamond-set design requirement
- Customer-selectable options on PDP:
  - Gold purity: 14Kt / 18Kt
  - Diamond grade: SI IJ / SI GH / VS GH / VVS EF
- Added a `Request This Piece` modal with:
  - Name
  - Phone
  - City
  - temporary client-side reference ID in `RG-######` format

### Data model updates

- Updated shared product typing to match the new business rules:
  - no gold-only pieces
  - fixed `goldWeightG`
  - fixed `diamondWeightCt`
  - `availableGoldPurities`
  - `availableDiamondGrades`
- Updated seeded catalogue data for all eight initial products to match this structure.

### Pricing logic

- Refactored pricing utilities so pricing can be calculated using injected live gold rates instead of only hardcoded defaults.
- Product cards now show rounded `Starting` prices when live rates are available.
- If live gold data is unavailable, cards and PDP pricing now fall back to `Price on request` instead of showing simulated metal pricing.

### Live gold price integration

- Added a server-side gold price service using Metals.dev.
- Added `/api/gold-price/current` route to expose the current snapshot to the frontend.
- Current implementation expects:
  - `GOLD_PRICE_PROVIDER=metalsdev`
  - `METALS_DEV_API_KEY`
  - `GOLD_PRICE_REFRESH_SECONDS`
- Important decision:
  - We removed fake reference-rate behavior for the active UI path.
  - If live data is not available, the UI should not pretend to have a real gold-linked price.
